# Plan A — Schema Extensions (events + user) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add three additive columns the Figma screens require — `stress_events.user_stress_level` (0–100 slider), `stress_events.mood_chips` (separate from triggers), and `users.display_name` ("Amy" on profile) — without changing existing behavior.

**Architecture:** Pure additive migration on Postgres. New columns are nullable; existing rows and existing API consumers keep working. SQLAlchemy models, Pydantic schemas, and the events router are extended in lockstep so the new fields round-trip on POST/GET/PATCH. The user-facing display_name is read-only on `GET /account/me` in this plan; setting it lands in Plan G.

**Tech Stack:** Python 3.12, FastAPI 0.136, SQLAlchemy 2.0 async, Alembic, Pydantic v2, asyncpg, pytest + pytest-asyncio + httpx.

---

## Decisions Locked

- `user_stress_level` lives on `stress_events`, alongside `model_confidence`. They are **not** the same value: `model_confidence` (0.0–1.0) is the on-device Mamba probability; `user_stress_level` (0–100) is the slider the user drags in the Stress Log screen.
- `mood_chips TEXT[]` is a separate array from `log_chips`. `log_chips` will be repurposed as **trigger chips** in Plan B (no rename needed yet — semantics shift when categories arrive). For now treat `log_chips` as opaque; just add `mood_chips` next to it.
- `display_name VARCHAR(64)` is nullable. No PATCH endpoint in this plan — Plan G owns profile mutations.
- No `category_id` FK in this plan. That column is added in Plan B together with the `trigger_categories` table so the migration is atomic.

@pytest.mark.asyncio
async def test_post_event_persists_user_stress_level_and_mood_chips(
    client: AsyncClient,
    db_session: AsyncSession,
    supabase_jwt_secret: str,  # noqa: ARG001
    auth_headers: Any,
    make_user: Any,
) -> None:
    me = await make_user()

    resp = await client.post(
        "/api/v1/events",
        headers=auth_headers(str(me.supabase_user_id)),
        json={
            "detected_at": datetime(2026, 5, 6, 9, tzinfo=UTC).isoformat(),
            "user_stress_level": 62,
            "mood_chips": ["anxious", "overwhelmed"],
            "logged": True,
        },
    )
    assert resp.status_code == 201, resp.text
    body = resp.json()
    assert body["user_stress_level"] == 62
    assert body["mood_chips"] == ["anxious", "overwhelmed"]

    refreshed = (
        await db_session.execute(
            select(StressEvent).where(StressEvent.id == uuid.UUID(body["id"]))
        )
    ).scalar_one()
    assert refreshed.user_stress_level == 62
    assert refreshed.mood_chips == ["anxious", "overwhelmed"]
